Script 03 — Methods

Normalized Amplitude Spectra

Each channel normalized to max 1, stacked with fixed vertical offsets for cross-channel comparison.

MATLABVisualization
03_normalized_amplitude_spectra.m
The Readability Problem

When all 64 channels' amplitude spectra are plotted without normalization, channels with high amplitude dominate and low-amplitude channels are invisible. The solution is to normalize each channel to a maximum of 1 and add a fixed vertical offset proportional to the channel index, creating a waterfall-style plot where all 64 channels are legible simultaneously.

Core Code
03_normalized_amplitude_spectra.mMATLAB
offsetStep = 0.1; % vertical spacing between channels for ch = 1:numChannels % ... compute FFT P1 as before ... % Normalize each channel to its own maximum P1_norm = P1 / max(P1); % Offset by channel index offset = (ch - 1) * offsetStep; plot(f, P1_norm + offset); end % Y-axis labels at offset positions yticks(0:offsetStep:(numChannels-1)*offsetStep);
Thought Process

The 0.1 offset step was chosen empirically. With 64 channels a step of 0.1 produces a total span of 6.3 units. The normalized spectra peak at 1.0, so there is minimal overlap between adjacent channels while still fitting the entire stack on a readable figure.

Normalizing per-channel is correct here because the goal is to compare frequency shape, not amplitude. Whether a channel has peak amplitude 1 µV or 50 µV is irrelevant for identifying which channels show beta activity during the task.
