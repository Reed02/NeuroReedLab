DTW Variant Match

Extends cross-correlation matching with Dynamic Time Warping and signal-variant generation to improve robustness to temporal variability in EEG.

Python MNE DTW Pattern Matching
20_dtw_variant_match.py
Why This Exists

Cross-correlation assumes signals are aligned in time, which is often unrealistic for EEG. Even when two recordings reflect the same cognitive state, neural responses can shift in latency due to reaction time differences, cortical variability, or preprocessing effects.

This script addresses that limitation by introducing Dynamic Time Warping (DTW), which allows elastic alignment of two signals before computing similarity. It also improves robustness further by comparing each signal against multiple realistic synthetic variants rather than a single reference trace.

Setup & Usage

This implementation uses Python with MNE and the fastdtw library for efficient approximate DTW computation. It is designed for the Eyes Open baseline subset of the EEG dataset from Zenodo and operates on EDF files only.

Install dependency: pip install fastdtw

As with previous scripts, file paths are manually specified and must be updated to match the local dataset directory.

def compute_similarity(sig1, sig2, method="dtw"):
  distance, _ = fastdtw(sig1, sig2, dist=scalar_dist)
  return 1 / (1 + distance)

def generate_variants(sig, num_variants=10, peak_shift_std=20, ...):
  peaks, _ = find_peaks(sig, distance=20)
  variants = []

  for _ in range(num_variants):
    variant = np.copy(sig)
    for peak in peaks:
      shift = int(np.random.normal(0, peak_shift_std))
      height_factor = 1 + np.random.normal(0, peak_height_std)
      width_factor = max(1 + np.random.normal(0, peak_width_std), 0.1)
      variant = deform_peak(variant, peak, shift, height_factor, width_factor)

  return variants
Interpretation

DTW improves over cross-correlation by aligning signals non-linearly in time before measuring distance. This makes it robust to latency shifts, variable reaction times, and temporal stretching—common properties of EEG signals across subjects.

The variant generator further increases robustness by modeling realistic EEG variability: each signal is transformed into multiple plausible versions with perturbed peak timing, amplitude, and width. A match is accepted if any variant yields a sufficiently small DTW distance.

Trade-off

This approach improves sensitivity by allowing flexible alignment across physiological variability, but it also increases the risk of false positives by expanding the acceptance space. In preprocessing pipelines, this is often an acceptable trade-off since removing valid EEG data is more damaging than retaining a small number of borderline samples.
