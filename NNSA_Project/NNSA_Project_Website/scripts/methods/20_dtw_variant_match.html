DTW Variant Match

Extends the cross-correlation matcher with true Dynamic Time Warping and a signal-variant generator to improve outlier detection on warped or time-shifted EEG recordings.

PythonMNEDTWPattern Matching

Improvement Over Script 19

Script 19 used cross-correlation, which requires signals to share the same temporal structure at the same time points. EEG from different subjects doing the same task can have the same spectral character but slightly different timing — one subject reacts 200 ms later, or a peak is broadened by different cortical geometry. DTW handles this by elastically aligning two sequences before computing distance, making it robust to those temporal shifts. Script 20 adds DTW and goes one step further: it generates synthetic variants of each signal and asks whether a target file matches any variant, not just the original.

Thought Process

The variant generator perturbs peak locations, heights, and widths independently for each peak in the signal. This models the realistic ways EEG waveforms vary across subjects: the same neural event at a slightly different latency, with a slightly different scalp projection. If a target file matches any one of 20 variants, it is accepted as a consistent member of the group.

The fastdtw library provides an O(N) approximation to full DTW, which is essential here — full DTW on 5000-sample EEG signals would be prohibitively slow for pairwise comparisons across 10 files times 20 variants each.

Trade-off

The variant-matching approach is more permissive than direct DTW comparison: a file can pass by matching just one synthetic variant rather than the original signal. This reduces false rejections of genuinely valid EEG but also risks accepting truly bad files. For outlier removal before training, false rejections are more costly than false acceptances, so the permissive direction is the right default.
