Script 08 — Methods

Extract Average FFT Features

Shift from visualization to ML: saves average FFT across 64 channels as compact .mat files.

MATLABFeature Extraction
08_extract_avg_fft_features.m
The Shift from Visualization to Machine Learning

Scripts 01–07 produced visualizations for human inspection. This script is the pivot to machine learning: instead of saving images, it saves compact numerical feature vectors that a CNN can learn from. The feature chosen is the average FFT amplitude spectrum across all 64 channels, computed after 14–50 Hz bandpass filtering and 100ms smoothing.

Each output .mat file contains two variables: avg_fft_row (the mean amplitude at each frequency bin) and f (the frequency axis). The EEG_CNN_Features_AvgFFT Zenodo dataset is the direct output.

Core Code
08_extract_avg_fft_features.mMATLAB
% Compute FFT for all channels, then average fft_all = zeros(nChannels, length(f)); for ch = 1:nChannels Y = fft(smoothed_data(ch,:)); P2 = abs(Y/N); P1 = P2(1:floor(N/2)+1); P1(2:end-1) = 2*P1(2:end-1); fft_all(ch,:) = P1; end avg_fft = mean(fft_all, 1); avg_fft_row = avg_fft(:)'; % ensure row vector save(saveFile, 'avg_fft_row', 'f');
Thought Process

Averaging across all 64 channels is a simplification chosen to keep feature dimensionality manageable for the first CNN experiments. It discards all spatial information about which brain regions show elevated activity. Saving both avg_fft_row and f in the same file makes the feature vector self-contained — you always know what frequency each element corresponds to.

Planned improvement

Averaging across channels discards all spatial information. A straightforward improvement would be to save the full 64×N FFT matrix instead, giving the CNN the opportunity to learn which channels are most discriminative per class. This is a documented next step.
