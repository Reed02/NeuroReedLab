Raw Data Plotting

Batch-loads all EDF files from a subject folder with MNE, converts the first recording to a PyTorch tensor, and opens MNE’s interactive signal browser.

PythonMNEPyTorchVisualization
09b_raw_data_plotting.py
Purpose

Before writing any analysis code, the first useful thing to do with a new EDF dataset is simply look at it. This script loads all EDF files from a single subject folder into a list of MNE Raw objects, converts the first recording into a PyTorch float32 tensor, prints its shape, and opens MNE’s interactive browser. It is the entry point for visually confirming that the files load correctly, that channel count and sample count are what the dataset documentation claims, and that the signal is not obviously corrupt before committing to a full preprocessing pipeline.

Core Code
09b_raw_data_plotting.pyPython
# Batch-load all EDF files from subject folder raw_files = [] for file_name in os.listdir(folder_path): if file_name.endswith('.edf'): raw = mne.io.read_raw_edf(file_path) raw_files.append(raw) # Pull first file into a PyTorch tensor to verify shape first_file = raw_files[0] data = first_file.get_data() # numpy array: (channels, samples) tensor_data = torch.tensor(data, dtype=torch.float32) print(tensor_data.shape) # e.g. torch.Size([64, 9760]) # Open MNE interactive signal browser first_file.plot(block=True)
Thought Process

The block=True argument on raw.plot() pauses script execution until the browser window is closed, which is necessary when running from the command line — without it the plot opens and immediately closes before you can interact with it. MNE’s browser lets you scroll through all 64 channels in time, zoom into frequency bands, and mark bad channels interactively, making it significantly more useful than a static matplotlib figure for initial data inspection.

Converting to a PyTorch tensor at this stage is a lightweight test of the full data path from EDF to model input. If the shape printed here doesn’t match what the CNN loaders expect — for example, a different number of channels than the 64 documented in the BCI2000 dataset — catching it here is far cheaper than debugging it later inside a training loop.

What to check in the browser

Channels Fp1, Fp2, AF7, and AF8 (frontal) will typically show the largest amplitude excursions due to eye movement and blink artifacts. Motor cortex channels C3, Cz, and C4 are the primary signals of interest for hand motor imagery. If any channel is flat (zero variance) or rail-clipped (constant at min/max), it should be excluded before analysis.
