DTW Pattern Match

Cross correlates ten baseline EEG files pairwise using normalized cross correlation to identify which recordings share a common signal pattern.

Python MNE Pattern Matching
19 dtw pattern match.py
Motivation

Before using a set of baseline recordings as a reference class for the CNN, it is worth asking: do these files actually look like the same thing? Ten subjects resting with eyes open should share broad spectral characteristics, but individual differences and recording artifacts can make some files outliers. This script compares every file to every other and flags those that fail to match at least half the group a lightweight outlier detector that requires no labeled ground truth.

Setup and Usage

No BioSig or MATLAB setup needed this is pure Python MNE. This script compares Eyes Open baseline recordings specifically, so it needs EDF files from EEG CNN Dataset EDF zip on Zenodo. The .set format dataset will not work here since it does not include the baselines.

The file paths list at the top of the script is hardcoded to ten specific files on one specific machine. Update these to point at the Eyes Open baseline files or any other set of files you would like to compare inside your own copy of the dataset.

Thought Process

Z score normalization before correlation is critical here. Without it, a subject with high amplitude EEG would always score high similarity against any other file simply because cross correlation scales with signal energy. Normalizing makes similarity reflect waveform shape rather than amplitude.

The normalized cross correlation peak divided by signal length is a rough proxy for cosine similarity of the two signals in time. It is not a rigorous DTW distance despite the script filename the DTW implementation comes in Script 20. It is fast and adequate for flagging gross outliers before training.

Naming note

This script is titled DTW in the filename but actually uses cross correlation. True Dynamic Time Warping is implemented in Script 20. The naming reflects the evolution of the approach during development rather than the final implementation.
