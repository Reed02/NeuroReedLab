Script 19 — Methods

DTW Pattern Match

Cross-correlates ten baseline EEG files pairwise using normalized cross-correlation to identify which recordings share a common signal pattern.

PythonMNEPattern Matching
19_dtw_pattern_match.py
Motivation

Before using a set of baseline recordings as a reference class for the CNN, it is worth asking: do these files actually look like the same thing? Ten subjects resting with eyes open should share broad spectral characteristics, but individual differences and recording artifacts can make some files outliers. This script compares every file to every other and flags those that fail to match at least half the group — a lightweight outlier detector that requires no labeled ground truth.

def compute_similarity(sig1, sig2, max_len=5000): # Truncate and z-score normalize both signals min_len = min(len(sig1), len(sig2), max_len) sig1 = (sig1[:min_len] - np.mean(sig1[:min_len])) / (np.std(sig1[:min_len]) + 1e-8) sig2 = (sig2[:min_len] - np.mean(sig2[:min_len])) / (np.std(sig2[:min_len]) + 1e-8) # Normalized cross-correlation peak as similarity score corr = np.correlate(sig1, sig2, mode='full') return np.max(corr) / min_len # Flag files matching ≥ half the group above threshold for i in range(n): matches = sum(1 for j in range(n) if i != j and compute_similarity(main_signals[i], main_signals[j]) >= similarity_threshold) if matches >= n // 2: pattern_indices.append(i)
Thought Process

Z-score normalization before correlation is critical here. Without it, a subject with high-amplitude EEG would always score high similarity against any other file simply because the cross-correlation magnitude scales with signal energy. Normalizing makes the similarity score reflect waveform shape rather than amplitude.

The normalized cross-correlation peak divided by signal length is a rough proxy for cosine similarity of the two signals in time. It is not a rigorous DTW distance (despite the script filename — the DTW implementation came in Script 20), but it is fast and adequate for flagging gross outliers before training.

Naming note

This script is titled DTW in the filename but actually uses cross-correlation. True Dynamic Time Warping is implemented in Script 20. The naming reflects the evolution of the approach during development rather than the final implementation.
