Topomap Prototype

Proof-of-concept with random placeholder data: validates geometry before using real PSDs.

PythonTopomap
05_topomap_prototype.py
Why Prototype First

Topomap generation involves several independent pieces: loading channel positions from CSV, interpolating scattered data onto a regular grid, masking outside the head circle, and rendering with the correct colormap and head outline overlay. Any one of these can have subtle bugs. Using random placeholder values validates the entire rendering geometry without any dependency on real data — if it looks wrong with random data the issue is definitely in the rendering, not the PSD computation.

# Load channel positions saved by script 11 positions_df = pd.read_csv(pos_file) x_pos = positions_df['X'].values y_pos = positions_df['Y'].values # Random placeholder values — one per channel psd_values = np.random.rand(len(channels)) # Cubic interpolation onto 200x200 grid grid_x, grid_y = np.mgrid[0:1:200j, 0:1:200j] grid_z = griddata((x_pos, y_pos), psd_values, (grid_x, grid_y), method='cubic') # Mask everything outside the head circle → renders transparent mask = (grid_x - 0.5)**2 + (grid_y - 0.5)**2 > 0.45**2 grid_z[mask] = np.nan
Thought Process

The 200×200 grid balances resolution and render time. A 100×100 grid is noticeably pixelated; a 400×400 grid takes 4× longer with minimal visual improvement. Setting masked pixels to np.nan rather than 0 lets matplotlib render them transparent, giving a clean head outline rather than a colored rectangular border.

The prototype confirmed all 64 channel positions from the CSV mapped inside the head outline, the cubic interpolation produced smooth gradients, and the head overlay rendered correctly. Only after all this was confirmed was real PSD data wired in.
