Script 02 — Data Prep

Folder-Level Segmentation

Wraps the single-file logic in a loop over all EDFs in one directory.

MATLABEEGLAB
02_segment_one_folder.m
What Changed

The core segmentation logic is identical to script 01. The only addition is a dir() call to find all .edf files in the current directory, and a loop that processes each one. For each file a subfolder named "Segment of [filename]" is created, keeping each file's segments together and easy to trace.

Core Code
02_segment_one_folder.mMATLAB
% Find all EDF files in the current directory edfFiles = dir(fullfile(edfFolder, '*.edf')); for f = 1:length(edfFiles) edfFile = edfFiles(f).name; [~, nameNoExt, ~] = fileparts(edfFile); % Create output subfolder for this file segFolder = fullfile(edfFolder, ['Segment of ', nameNoExt]); if ~exist(segFolder, 'dir'), mkdir(segFolder); end % ... same segmentation logic as script 01 ... end
Thought Process

Keeping each EDF's segments in its own named subfolder avoids the flat-folder problem: with hundreds of segments per run across 14 runs, a flat structure is unmanageable. The subfolder name includes the original filename so any segment can always be traced back to its source EDF.

This script was tested on one subject folder before proceeding to the full batch version. Once it produced the correct number of segments with sensible filenames, scaling was just a matter of changing the loop to cover all subjects.

Intermediate step

This script processes one subject folder. Script 03 loops over all 109 subjects automatically. This version is still useful for debugging or reprocessing a single subject without running the full batch.
