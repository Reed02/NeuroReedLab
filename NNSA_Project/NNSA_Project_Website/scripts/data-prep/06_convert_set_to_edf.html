Script 06 — Data Prep

Convert .set Files to EDF

Converts classified EEGLAB .set/.fdt files to .edf for Python MNE compatibility.

MATLABFormat Conversion
06_convert_set_to_edf.m
Why Conversion Was Needed

EEGLAB's native .set format is ideal for MATLAB-based analysis but Python's MNE library works primarily with EDF. All Python analysis scripts use mne.io.read_raw_edf(). Rather than rewriting those scripts to handle .set files, converting the classification output to EDF was the cleaner and faster path.

Channel Label Cleanup
06_convert_set_to_edf.mMATLAB
% EDF spec: channel labels must be ≤16 chars and have no whitespace for ch = 1:EEG.nbchan lbl = EEG.chanlocs(ch).labels; lbl = regexprep(lbl, '\s+', ''); % strip whitespace lbl = lbl(1:min(16, end)); % truncate to 16 chars EEG.chanlocs(ch).labels = lbl; end EEG = eeg_checkset(EEG); % Write EDF with 1-second record blocks pop_writeeeg(EEG, outputEDF, 'TYPE', 'EDF', 'recordlength', 1);
Thought Process

The EDF format has a strict 16-character limit on channel labels. Some EEGLAB channel names include spaces or exceed 16 characters, which causes pop_writeeeg to either error or silently corrupt the label. The preprocessing step strips whitespace with a regex and truncates to 16 characters before the write.

The recordlength of 1 second is the standard EDF record length. It is compatible with all EDF readers including MNE. The output mirrors the EEG_Classification/ folder structure: 9 subfolders with identical class names. The EEG_Classification_EDFFormat Zenodo dataset is the direct output.
