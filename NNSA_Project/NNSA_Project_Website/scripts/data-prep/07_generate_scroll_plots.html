Script 07 — Data Prep

Generate EEGLAB Scroll Plots

Visual quality check: renders and saves a scroll-view PNG for every file in every class folder.

MATLABEEGLABVisualization
07_generate_scroll_plots.m
Purpose

After building a large automated pipeline it is easy to have subtle errors — files routed to the wrong class, truncated segments, channels with flat or clipped signals — that produce no errors but generate bad data. The only reliable way to catch these is visual inspection. This script uses EEGLAB's pop_eegplot to render the standard EEG scroll view for every file across all task folders and saves each as a high-resolution PNG.

Core Code
07_generate_scroll_plots.mMATLAB
for i = 1:length(taskFolders) edfFiles = dir(fullfile(srcFolder, '*.edf')); for j = 1:length(edfFiles) EEG = pop_biosig(fullfile(srcFolder, edfFile)); figure(figNum); clf; pop_eegplot(EEG, 1, 1, 1); pause(2); % wait for async GUI render exportgraphics(gcf, outFile, 'Resolution', 300); close(gcf); end end
Thought Process

The pause(2) call is necessary because pop_eegplot renders an interactive EEGLAB GUI figure asynchronously. Without a pause, exportgraphics can capture a blank or partially-rendered window. Two seconds was sufficient on the development machine; on a slower system this may need to be increased.

This script processes the EDF files (output of script 06) rather than the .set files because it was written after the conversion step. The output images are saved into a mirrored folder structure under images/ on the Desktop, one subfolder per task class.

Manual inspection strategy

The goal is not to look at every image individually, but to scroll through them quickly to spot obvious anomalies: all-zero channels, extreme amplitude spikes, completely flat segments, or misrouted files. Most images should look like normal EEG scroll plots. Any that look dramatically different warrant investigation.
