Script 03 — Data Prep

All-Subjects Batch Processing

Full batch pipeline: all 109 subject folders processed in one run into a master output folder.

MATLABEEGLABBatch
03_segment_all_subjects.m
What This Does

This is the production version of the segmentation pipeline. It runs from the directory containing all subject folders (S001 through S109), finds every folder matching S*, and applies the segmentation logic to every EDF within each. All output lands in a single master EEG_Segments/ folder on the Desktop with one subfolder per EDF file.

On the full dataset this produces thousands of subfolders containing tens of thousands of .set and .fdt pairs. The EEG_Segments_SetFormat dataset on Zenodo is the direct output of this script — if you download it you can skip this script entirely.

Core Code
03_segment_all_subjects.mMATLAB
% Master output on Desktop masterFolder = fullfile(getenv('HOME'), 'Desktop', 'EEG_Segments'); if ~exist(masterFolder, 'dir'), mkdir(masterFolder); end % Loop: subjects, then EDFs within each subject subjectFolders = dir(fullfile(baseFolder, 'S*')); subjectFolders = subjectFolders([subjectFolders.isdir]); for s = 1:length(subjectFolders) subjectPath = fullfile(baseFolder, subjectFolders(s).name); edfFiles = dir(fullfile(subjectPath, '*.edf')); for f = 1:length(edfFiles) % ... segmentation logic ... fprintf('Done: %s\n', segmentFolderPath); end end
Thought Process

The two-level loop mirrors the dataset structure exactly. Writing all outputs to a flat list inside EEG_Segments/ rather than replicating the subject/run hierarchy makes the output easier to scan and count. The subject ID is still encoded in every filename so nothing is lost.

The fprintf progress logging after each file is important on a long batch job. Without it there is no feedback during the hours it takes to process the full dataset, making it impossible to tell if the script is hanging or just slow.
