Script 01 — Data Prep

Single-File Segmentation

Proof-of-concept: parse one event file, slice one EDF into labeled T0/T1/T2 segments.

MATLABEEGLAB
01_segment_single_file.m
Background

The BCI2000 dataset stores each recording run as a continuous EDF file. Before any analysis is possible, these need to be cut at event boundaries. Each EDF has a companion .edf.event text file listing every event marker with its label (T0, T1, or T2) and its duration in seconds. T0 marks rest periods; T1 and T2 mark specific motor or imagined motor events depending on the run number.

The approach here was to hardcode a single file — S001R03.edf — and get every detail right before scaling up. A mistake in the sample math would corrupt every output file silently.

Core Code
01_segment_single_file.mMATLAB
% Open event file and read as raw text fid = fopen(filename, 'r'); rawText = fread(fid, '*char')'; fclose(fid); % Regex captures label (T0/T1/T2) and duration in seconds pattern = '(T[0-2])\s*duration:\s*([0-9]+\.[0-9]+)'; matches = regexp(rawText, pattern, 'tokens'); % Convert durations to samples, handle last-segment overflow segment_lengths_samples = round(durations * fs); if sum(segment_lengths_samples) > total_samples segment_lengths_samples(end) = total_samples - sum(segment_lengths_samples(1:end-1)); end % Slice and save each segment as .set EEG_seg.data = EEG.data(:, start_idx:end_idx); EEG_seg.pnts = size(EEG_seg.data, 2); pop_saveset(EEG_seg, 'filename', fullpath);
Thought Process

The regex pattern is intentionally permissive about whitespace between the label and the duration keyword, since the event file format is not perfectly consistent across all subjects in the dataset. Each regex match returns a two-element cell: the label string and the duration string, which is then converted with str2double.

The overflow check on the last segment is essential. Floating-point rounding when converting durations to sample counts means the sum can exceed the actual recording length by one or two samples. Without the correction, MATLAB errors trying to index beyond the end of the data matrix.

Segment filenames encode everything needed to identify them: S001R03_segment_002_T1.set gives subject, run, sequence number, and event label. This makes the downstream sorting step straightforward.

Why start with one file

A bug in the regex or sample math produces silently wrong outputs across all 109 subjects with no easy way to detect it afterward. Getting every edge case confirmed on one file takes 10 minutes. Reprocessing 109 subjects takes hours.
