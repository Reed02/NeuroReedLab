Script 01 — Data Prep

Single File Segmentation

Proof of concept: parse one event file, slice one EDF into labeled T0/T1/T2 segments.

The approach here was to hardcode a single file — S001R03.edf — and get every detail right before scaling up.

% Convert durations to samples, handle last segment overflow

Each regex match returns a two element cell: the label string and the duration string, which is then converted with str2double.

The overflow check on the last segment is essential. Floating point rounding when converting durations to sample counts means the sum can exceed the actual recording length by one or two samples.

Segment filenames encode everything needed to identify them: S001R03_segment_002_T1.set gives subject, run, sequence number, and event label.
