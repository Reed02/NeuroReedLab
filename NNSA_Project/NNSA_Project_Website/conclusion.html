10-Run Evaluation — Results & Reflections

What the Data Actually Shows

Each CNN script was run 10 independent times with identical hyperparameters and no fixed random seed. The results below replace any single-run numbers from earlier in the project and represent the honest picture of what these architectures can and cannot do on this dataset.

02 — Spectrogram Binary: Eyes Open vs. Closed (val acc) 50% 65.9% 78.4% 95.5% ⚠ Val acc only — needs proper test split

What Is Actually Working
Confirmed Results
  • Binary CNN (Scripts 01 & 02): 70.2% and 78.4% mean — both are 20+ points above the 50% chance baseline and consistent across runs
  • Full segmentation and labeling pipeline runs end-to-end on all 109 subjects
  • FFT feature extraction (MATLAB → .mat → Python) pipeline works correctly
  • Topographic frequency-band maps generate correctly and show interpretable alpha lateralization
  • PSD comparison shows Eyes Closed and Rest separating cleanly in 8–12 Hz band
  • All processed data publicly accessible on Zenodo
Confirmed Problems
  • 11-class raw EDF: 17.6% mean — near 9.1% chance, shallow 1D CNN cannot learn motor class distinctions from raw time-series
  • 11-class FFT at 50 epochs: 20.5% mean — the architecture is systematically undertrained; the earlier 60.23% single run was at 100 epochs and is plausibly real but unverified
  • High initialization sensitivity in binary scripts: 29-point spread (50–79.6%) without a fixed seed
  • Spectrogram early stopping is unstable: best checkpoint captured at epoch 1 in some runs, measuring initial weights not learned features
  • No artifact rejection applied before CNN training
  • No fixed random seed — results not reproducible run-to-run
Priority Next Steps
  • Fix random seed in all CNN scripts to make results reproducible
  • Retest Script 04 at 150 epochs — the 3-point spread across 10 runs at 50 epochs means every run hits the same wall before convergence; more training time is the obvious fix
  • Add a proper test split to the spectrogram CNN so reported accuracy is on held-out data, not the val checkpoint
  • Apply bandpass filtering and artifact threshold rejection before CNN input
  • Investigate EEGNet or ShallowConvNet for the 11-class problem — purpose-built EEG architectures with depthwise separable convolutions are the correct tool here
  • Explore per-band power (delta, theta, alpha, beta) as CNN input rather than raw time-series or averaged FFT
Script-by-Script Analysis

Scripts 01 & 02 — Binary Classification: The Confirmed Results

Both binary scripts classify Eyes Open vs. Eyes Closed and both work. Script 01 (raw 1D CNN) reaches 70.2% mean; Script 02 (2D spectrogram CNN) reaches 78.4% mean val accuracy. The physiological reason is straightforward: Eyes Closed powerfully suppresses posterior alpha activity (~8–13 Hz), a large and well-documented signal that even a shallow CNN reliably detects. The 8-point gap between scripts suggests that giving the network explicit frequency structure via spectrogram conversion is helpful, though the comparison is not perfectly clean since Script 02 reports val accuracy rather than held-out test accuracy. The dominant problem in both scripts is initialization sensitivity — without a fixed seed, a single bad initialization collapses binary accuracy to 50% (chance). Fixing the seed and running a three-way split would make both results fully rigorous.

Scripts 03 & 04 — 11-Class Classification: What These Results Mean

Both 11-class scripts run the same 11 motor categories (Rest, Eyes Open, Eyes Closed, Real Left/Right Fist, Imagined Left/Right Fist, Real/Imagined Both Fists, Real/Imagined Both Feet). Script 03 uses raw EDF time-series; Script 04 uses precomputed average-FFT vectors. Both reach 17–20% mean accuracy vs. 9.1% chance — a real margin above chance but far from useful classification.

The key distinction between the two scripts is why they underperform. Script 03 is architecturally wrong for this input — a 2-layer 1D CNN cannot learn frequency-selective filters for 11-class motor discrimination from raw EEG. Script 04 is architecturally plausible but undertrained: the 3.21-point spread across all 10 runs (19.3–22.5%) is the signature of a model that hits the same local minimum every time because it hasn’t trained long enough to escape it. The earlier 60.23% single run at 100 epochs, combined with that run’s loss curve showing the model just beginning to improve at epoch 70, makes a strong case that Script 04 needs 150+ epochs to converge. That test has not yet been done systematically.

The Most Important Correction From 10-Run Testing

Before these repeated runs, Script 04 was described as the best-performing CNN based on a single result of 60.23%. That description was wrong in a specific way: the script was run at 100 epochs that one time, while all 10 repeated runs used 50 epochs. The 10-run mean of 20.5% does not contradict the earlier result — it reveals that the architecture needs at least 100 epochs to produce that result, and 50 epochs is systematically insufficient. The correct next experiment is 10 runs of Script 04 at 150 epochs with a fixed seed. If the mean comes back near 60%, that confirms FFT features are the right approach for this problem. If it comes back near 30%, the features themselves are the limitation.

Broader Context

Motor imagery EEG classification on the PhysioNet BCI2000 dataset is a well-studied benchmark. Published multi-class results at the high end reach 60–80%, using subject-specific calibration, artifact rejection, and purpose-built architectures. The current binary results at 70–78% mean already sit within the published range for the simplest version of the problem. The 11-class gap between 17–20% and published benchmarks is real but not a fundamental data problem — it is an architecture and training budget problem, both of which are tractable.

If 11-class accuracy can be pushed to 60–70% consistently, the downstream applications are meaningful: kinesiological VR simulation and robotic control through imagined motor commands decoded from non-invasive scalp EEG. The binary result already demonstrates that the core pipeline — raw BCI2000 recordings through MATLAB segmentation to PyTorch classification — produces real signal. Building from here toward reliable multi-class performance is the logical next phase of this project.
