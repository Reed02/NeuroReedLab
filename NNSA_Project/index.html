Decoding Motor Imagery
from EEG Signals

DatasetPhysioNet BCI2000 · OpenNeuro
Subjects109 participants
Channels64-channel EEG
GoalCNN motor imagery classification
ToolsMATLAB · EEGLAB · Python · MNE

This project sets out to classify motor imagery from scalp EEG — distinguishing whether a person is performing or imagining a physical movement — using a CNN trained on a large open-source dataset.

The dataset (Schalk et al., BCI2000, 2004) provides 109 subjects completing 14 runs each: two baselines and four tasks involving real and imagined hand and foot movements, recorded in BIDS-compliant EDF format.

Work proceeded in two phases: a MATLAB pipeline to segment, organize, and explore the raw EDF recordings, followed by a Python phase for spectral analysis, topographic brain mapping, and CNN experimentation.

All source files live on GitHub at Reed02/NeuroReedLab under NNSA_Project/. Every section below links directly to the relevant file.

The core challenge: 64 channels, 109 subjects, 14 run types, continuous recordings — all needing to be parsed, labeled, and transformed into something a neural network can learn from.

Phase 1 — MATLAB Segmentation Pipeline
01
Segmentation · Proof of Concept

Single-File Segmentation

01_segment_single_file.m
MATLABEEGLAB

Everything starts with one hardcoded file — S001R03.edf — to nail the segmentation logic before scaling. Each continuous EDF has a paired .edf.event text file listing T0/T1/T2 labels and their durations in seconds. A regex parses the event file, durations convert to samples, and the EEG matrix is sliced and saved as individual .set files.

01_segment_single_file.mMATLAB
% Parse event file — label + duration pattern = '(T[0-2])\s*duration:\s*([0-9]+\.[0-9]+)'; matches = regexp(rawText, pattern, 'tokens'); % Duration (s) → samples, then slice segment_lengths_samples = round(durations * fs); EEG_seg.data = EEG.data(:, start_idx:end_idx);
Why start with one file

Getting the regex, sample math, and edge cases right on a single file avoids running broken logic across all 109 subjects. Once confirmed, generalization is just a loop.

02
Segmentation · Folder Level

Folder-Level Segmentation

02_segment_one_folder.m
MATLABEEGLAB

The single-file logic is wrapped in a loop over all .edf files in the current directory. For each, a named subfolder "Segment of [filename]" is created and filled with .set files — one per event. Filenames encode both sequence number and label (e.g., S001R03_segment_002_T1.set).

Organisation · Classification

Sorting Files into Classification Folders

05_sort_into_class_folders.m
MATLABClassification

With counts verified, this script copies segments into a final EEG_Classification/ folder with 9 named subfolders. T0 always → Rest; T1/T2 routed by run number.

Discovery: class imbalance

Baseline folders ended up with only 109 files each (one per subject) while movement folders held 2,000–18,000. This imbalance later severely constrained CNN training for the multi-class problem.

MATLAB — Signal Exploration
01
Signal Exploration

Time-Domain Exploration

01_time_domain_exploration.m
MATLABVisualization

The first look at the actual signal. A single segment is loaded, bandpass filtered (1–50 Hz, 4th-order Butterworth), then smoothed with a 100ms moving average. Three plots are produced: original, filtered, and smoothed — all 64 channels overlaid. This established visual intuition for what clean EEG looks like before frequency-domain work began.

02
Signal Exploration

FFT and Peak Frequency Per Channel

02_fft_peak_frequency.m
MATLABFFT

High-pass filtered at 8 Hz to suppress slow drift, then FFT'd per channel. Amplitude spectra are normalized per channel and stacked with vertical offsets for readability. The script tracks the global peak frequency across all channels — the first attempt to identify which frequency range carries the most signal during motor tasks.

03
Signal Exploration

Normalized Amplitude Spectra

03_normalized_amplitude_spectra.m
MATLABVisualization

Each channel's FFT is normalized to a maximum of 1 then plotted with a fixed vertical offset. Makes it possible to compare all 64 channels' frequency profiles simultaneously without any single channel's amplitude dominating the display.

04
Signal Exploration

Sliding-Window Dominant Frequency Over Time

04_sliding_window_frequency.m
MATLABTime-Frequency

A 1-second window slides across each channel in 100ms steps. At each position, the dominant frequency within 8–80 Hz is extracted via FFT. The result is a temporal trace of dominant frequency per channel — showing how brain rhythms shift over the course of a task, not just their static average.

Batch Processing

Batch Image Export — 3 Plot Types

05_batch_image_export_3plots.m
MATLABBatch

First batch image pipeline. Loops over a folder of .set files and saves three PNGs per file (original, 1–50 Hz filtered, smoothed) into organized subfolders. The first attempt to visually inspect whether different tasks look distinct.

The 14–50 Hz range was chosen deliberately to focus on bands most associated with motor cortex activity.

Signal Quality · Python

Artifact Detection Across Subjects

01_artifact_detection.py
PythonMNE

The first Python script. MNE loads 6 baseline EDF files (Eyes Open/Closed, subjects 1–3), applies 1–40 Hz bandpass, Gaussian smoothing, and a large moving-average envelope. Artifacts are flagged as signal excursions beyond 3 standard deviations. Results display stacked or overlapping — a comparative quality check before heavier analysis.

Spectral Analysis · Python

PSD Comparison Across Movement Categories

04_psd_comparison.py
PythonMNEPSD

Welch PSD is computed across all 11 movement categories, normalized to dB. A percentage-match algorithm (15% tolerance) filters outlier files within each category before averaging. Matched averages are plotted overlapping to test whether different movements have distinct frequency signatures.

13
Topographic Mapping · Prototype

Topomap Prototype with Placeholder Data

05_topomap_prototype.py
PythonTopomap

A proof-of-concept with random placeholder PSD values. Loads channel positions from CSV, interpolates values onto a 200×200 grid using cubic interpolation, masks outside the head circle, and renders a heatmap per movement type. Validated the rendering geometry and color scaling before connecting real computed data.

14
Topographic Mapping · Full Pipeline

Full Frequency-Band Topomap Pipeline

06_topomap_frequency_bands.py
PythonMNETopomap

The complete topographic analysis. Welch PSD is computed per channel across all files in each movement folder. Mean power in each canonical band (delta 0.5–4, theta 4–8, alpha 8–13, beta 13–30, gamma 30–79 Hz) is extracted per channel, normalized, and interpolated onto the head grid. One figure per band shows all 11 movement conditions simultaneously — the most spatially informative output of the project.

06_topomap_frequency_bands.pyPython
FREQ_BANDS = { "Delta (0.5-4)": (0.5,4), "Theta (4-8)": (4,8), "Alpha (8-13)": (8,13), "Beta (13-30)": (13,30), "Gamma (30-79)": (30,79), } # Cubic interpolation onto 200×200 grid, mask outside head grid_z = griddata((x_pos, y_pos), values, (grid_x, grid_y), method='cubic') grid_z[(grid_x-.5)**2+(grid_y-.5)**2 > .45**2] = np.nan
15
Algorithm Validation · Python

Pattern Matching Algorithm Validation

07_pattern_match_validation.py
PythonAlgorithm Validation

25 synthetic datasets in a 5×5 matrix — each a noisy variant of the same U-curve, with noise increasing down rows. The percentage-match function (15% deviation tolerance) is tested: clean rows match easily, noisier rows fail gracefully. Confirmed correct behavior before trusting the algorithm on real EEG data in 04_psd_comparison.py.

Why this mattered

The matching logic filters EEG files before PSD averaging. Validating on synthetic data with known ground truth gave confidence in the 15% threshold before applying it to 50,000+ real segments.

Current Status & Reflections

Where the Project Stands

Classification TaskAccuracyNotes
Eyes Open vs. Eyes Closed (binary)70–80%Consistent across runs
Full 9-class motor imagery10–20%Near-random; some pattern present but not useful
What's Working
  • Binary CNN (eyes open/closed) reaches 70–80% consistently
  • Full segmentation and classification pipeline is solid
  • Topographic frequency-band maps generating correctly
  • PSD matching algorithm validated on synthetic data
  • Large dataset with room for extensive experimentation
Known Problems
  • 9-class CNN accuracy 10–20%, essentially random
  • Class imbalance: 109 baseline files vs. 2,000–18,000 movement segments
  • Baseline segments 56s longer than movement segments — temporal mismatch
  • Test file accidentally drawn from training set — earlier results invalid
Next Steps
  • Strictly separate training and test splits before any further CNN evaluation
  • Address class imbalance via augmentation, oversampling, or weighted loss
  • Trim baseline segments to match movement segment duration
  • Explore reinforcement learning to reduce reliance on direct pattern matching
  • Apply rigorous scientific method: hypothesis → experiment → analysis → conclusion
  • Consider richer features: wavelets, ERD/ERS, per-band power rather than raw FFT

On the 10–20% Result

A 10–20% accuracy on a 9-class problem sits just above chance (11%). The result is "not random enough for there to be no pattern" — the CNN is finding something, but whatever it's learning is too weak, too noisy, or poorly represented by the current feature set. Three factors are likely culpable: class imbalance biasing predictions toward movement classes, the 56-second baseline length mismatch giving baselines a fundamentally different temporal structure, and the accidental training/test overlap meaning none of the earlier multi-class numbers can be trusted.

What the Pipeline Gets Right

The data preparation work — raw BIDS EDF through segmentation, run-number-to-task mapping, classification folders, and EDF reconversion — is solid and reproducible. The binary classifier at 70–80% is a legitimate result: eyes-open vs. eyes-closed has well-understood EEG signatures (alpha suppression during open, strong posterior alpha during closed) and the CNN is clearly capturing something real. The topographic visualization pipeline also works correctly and produces interpretable per-band brain maps.

Broader Context

Motor imagery EEG classification is genuinely hard. State-of-the-art methods on this PhysioNet benchmark typically achieve 60–80% on the full multi-class problem with careful preprocessing, artifact rejection, and tuned architectures. The current pipeline uses no artifact rejection, no subject-specific normalization, and a relatively simple CNN — there is substantial headroom before concluding the approach is fundamentally flawed. If multi-class accuracy reaches 80–90% consistently, the application space includes VR kinesiological simulation and robotics, where imagined motor commands decoded from scalp EEG could enable real brain-machine interaction.
