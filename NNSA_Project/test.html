Decoding Motor Imagery
from EEG Signals

DatasetPhysioNet BCI2000 · OpenNeuro
Subjects109 participants
Channels64-channel EEG
GoalCNN motor imagery classification
ToolsMATLAB · EEGLAB · Python · MNE

This project sets out to classify motor imagery from scalp EEG recordings — distinguishing whether a person is performing or imagining a physical movement — using a convolutional neural network trained on a large open-source dataset.

The dataset comes from the BCI2000 system (Schalk et al., 2004), published on OpenNeuro with 109 subjects completing 14 runs each: two baselines and four tasks involving real and imagined hand and foot movements.

Work proceeded in two phases: a MATLAB pipeline to segment, organize, and explore the raw EDF recordings, followed by a Python phase for richer spectral visualization, topographic brain mapping, and CNN experimentation.

All source files are available in the GitHub repository, organized in the order they were written. Links to each file appear throughout this log.

The core challenge: 64 channels, 109 subjects, 14 run types, continuous recordings — all needing to be parsed, labeled, and transformed into something a neural network can learn from.

Phase 1 — MATLAB Segmentation Pipeline
01
Segmentation · Proof of Concept

Single-File Segmentation

01_segment_single_file.m
MATLABEEGLAB

Everything starts with a single hardcoded file — S001R03.edf — to nail the segmentation logic before scaling. Each continuous EDF has a paired .edf.event text file listing T0/T1/T2 labels and their durations in seconds. A regex parses the event file, durations are converted to samples, and the EEG matrix is sliced and saved as individual .set files.

01_segment_single_file.mMATLAB
% Parse event file — label + duration pattern = '(T[0-2])\s*duration:\s*([0-9]+\.[0-9]+)'; matches = regexp(rawText, pattern, 'tokens'); % Duration (s) → samples, then slice segment_lengths_samples = round(durations * fs); EEG_seg.data = EEG.data(:, start_idx:end_idx);
Why start with one file

Getting the regex, sample math, and edge cases right on one file avoids running broken logic across all 109 subjects. Once confirmed, generalization is a loop.

Folder-Level Segmentation

02_segment_one_folder.m

Sorting Files into Classification Folders

Discovery: class imbalance

Baseline folders ended up with 109 files each (one per subject), while movement folders held 2,000–18,000 segments. This imbalance later constrained CNN training and significantly hurt multi-class accuracy.

MATLAB — Signal Exploration
01
Signal Exploration

Time-Domain Exploration

01_time_domain_exploration.m
MATLABVisualization

The first look at the actual signal. A single segment is loaded, bandpass filtered (1–50 Hz, 4th-order Butterworth), then smoothed with a 100ms moving average. Three plots are produced: original, filtered, and smoothed — all 64 channels overlaid. This established visual intuition for what clean EEG looks like before frequency-domain work began.

02
Signal Exploration

FFT and Peak Frequency Per Channel

02_fft_peak_frequency.m
MATLABFFT

High-pass filtered at 8 Hz to suppress slow drift, then FFT'd per channel. The amplitude spectrum is normalized per channel and plotted with vertical offsets for readability. Tracks and reports the global peak frequency and which channel produced it — the first attempt to identify which frequency range carries the most signal during motor tasks.

03
Signal Exploration

Normalized Amplitude Spectra

03_normalized_amplitude_spectra.m
MATLABVisualization

A cleaner version of the spectral plot — each channel normalized to max 1 and stacked with a fixed vertical offset. Makes it possible to compare frequency profiles across all 64 channels simultaneously without any single channel's amplitude dominating.

04
Signal Exploration

Sliding-Window Dominant Frequency Over Time

04_sliding_window_frequency.m
MATLABTime-Frequency

A 1-second window slides across each channel in 100ms steps. At each position, the dominant frequency within 8–80 Hz is extracted from an FFT. The result is a temporal trace per channel showing how brain rhythms shift over the task — not just a static average.

Signal Quality · Python

Artifact Detection Across Subjects

01_artifact_detection.py
PythonMNE

The first Python script. MNE loads 6 baseline EDF files (Eyes Open/Closed, subjects 1–3), applies 1–40 Hz bandpass, Gaussian smoothing, and a large moving-average envelope. Artifacts are flagged when signal exceeds 3 standard deviations. Results displayed stacked or overlapping per subject.

01_artifact_detection.pyPython
norm = (smoothed - np.mean(smoothed)) / np.std(smoothed) ma = np.convolve(norm, np.ones(SMOOTH_WINDOW)/SMOOTH_WINDOW, 'same') artifact_mask = np.abs(ma) > THRESHOLD_FACTOR * np.std(ma)

15
Algorithm Validation · Python

Pattern Matching Algorithm Validation

07_pattern_match_validation.py
PythonAlgorithm Validation

25 synthetic datasets in a 5×5 matrix — each a noisy variant of the same U-curve, with noise increasing down the rows. The percentage-match function (15% tolerance) is tested: clean rows match easily, noisier rows fail gracefully. Confirmed the algorithm behaves correctly before trusting it to filter real EEG data.

Why this mattered

The same logic filters EEG files in 04_psd_comparison.py before PSD averaging. Validating on synthetic data with known ground truth gave confidence in the threshold before applying it to 50,000+ real segments.

Current Status & Reflections

Where the Project Stands

What's Working
  • Binary CNN (eyes open/closed) reaches 70–80% consistently
  • Full segmentation + classification pipeline is solid
  • Topographic frequency-band maps generating correctly
  • PSD matching algorithm validated on synthetic data
  • Large dataset with room for extensive experimentation
Known Problems
  • 9-class CNN accuracy 10–20%, essentially random
  • Class imbalance: 109 baseline files vs. 2,000–18,000 movement segments
  • Baseline segments 56s longer than movement segments — temporal mismatch
  • Test file accidentally drawn from training set — earlier results invalid
Next Steps
  • Strictly separate training and test splits before any further CNN evaluation
  • Address class imbalance — augmentation, oversampling, or weighted loss
  • Trim baseline segments to match movement segment duration
  • Explore reinforcement learning to reduce reliance on direct pattern matching
  • Adopt rigorous scientific method: hypothesis → experiment → analysis → conclusion, documented
  • Consider richer features beyond average FFT: wavelets, ERD/ERS, per-band power

On the 10–20% Result

A 10–20% accuracy on a 9-class problem sits just above chance (11%), which is frustrating but informative. The result is "not random enough for there to be no pattern" — the CNN is finding something, but whatever it is learning is too weak, too noisy, or poorly represented by the current feature set to be useful.

Three factors are likely culpable. First, the class imbalance means the network sees vastly more motor imagery examples than baselines during training, biasing predictions. Second, the 56-second length mismatch means baseline segments have a fundamentally different temporal structure than the 4-second movement segments. Third, the accidental overlap between training and test sets means the CNN had already seen the test data — none of the earlier multi-class accuracy numbers can be trusted.

What the Pipeline Gets Right

The data preparation work — from raw BIDS EDF through segmentation, run-number-to-task mapping, classification folders, and EDF reconversion — is solid and reproducible. The binary classifier reaching 70–80% is a legitimate result: eyes-open vs. eyes-closed has well-understood EEG signatures (alpha suppression during open, strong posterior alpha during closed), and the CNN is clearly capturing something real.

The topographic visualization pipeline also works correctly and produces interpretable per-band brain maps — useful for exploratory analysis even if not yet connected to the CNN training loop.

Broader Context

Motor imagery EEG classification is genuinely hard. State-of-the-art methods on this specific PhysioNet benchmark typically achieve 60–80% on the full multi-class problem with careful per-subject preprocessing, artifact rejection, and tuned architectures. The current pipeline uses no artifact rejection, no subject-specific normalization, and a relatively simple CNN. There is substantial headroom before concluding the approach is fundamentally flawed.

If multi-class accuracy can be pushed to 80–90% consistently, the application space includes VR kinesiological simulation and robotics — systems where imagined motor commands decoded from scalp EEG could enable meaningful brain-machine interaction.
